# Obsidian + Hermes Agent 完全体：会思考、会记忆、自动运行的个人操作系统

2026-06-10 12:52·[代码拆解师](https://www.toutiao.com/c/user/token/MS4wLjABAAAAy820T5mkijJZKsxtdysnZcpCjkesXd87y59h0kW_wn9hPFpf-SLsQsG2f33Ql98m/?source=tuwen_detail)

大多数个人知识管理系统，有一个致命问题：记忆。

你的 Obsidian 笔记存下了你知道的一切，但如果你不去主动翻它，它什么都不会告诉你。你的 AI 助手能执行各种任务，但会话一结束，它就什么都忘了。

把这两者连起来，问题彻底消失。

Obsidian 成为永久知识层。每一条笔记、每一个决定、每一个灵感、每一次项目更新，都以纯文本形式永远存在。

Hermes Agent 成为智能层。它读取你的知识库，跨笔记推理，执行工作流，自动把新的知识写回你的笔记系统。

它们合在一起，形成了一个能用你的积累思考、跨会话记住一切、在你睡觉时自动运行关键工作流的系统。

这篇文章基于 @cyrilXBT 的完整搭建指南和 @DamiDefi 的实测体验，整理出一份从零到一的完整方案。

#  一、为什么是这两个工具的组合

单独用 Obsidian：它是最好的知识存储系统。纯文本、本地存储、永久保存、可搜索、可链接。但它是被动的。你放进去什么，它存什么。它不会读你的笔记然后主动提炼洞察，不会自动执行工作流，不会从你的积累中主动产出内容。

单独用 Hermes：它是最好的自主 Agent 框架。持久记忆、技能执行、定时自动化、MCP 连接真实工具。但它的记忆存在 SQLite 数据库里，人类不可读，你不能打开它浏览或直接编辑，更没法像 Obsidian 那样构建丰富的知识网络。

组合起来的效果：

Obsidian 负责人类可读的永久存储。你读它、写它、链接笔记、构建多年积累的知识网络。

Hermes 通过 Filesystem MCP 直接读取你的 Obsidian 笔记库，用 Claude 跨所有笔记推理，执行自动化工作流产出新知识，把输出作为新笔记写回你的笔记库。

你的笔记库不再只是存储东西的地方，它是 Hermes 运行的操作环境。你写的每条笔记都成为 Hermes 使用的上下文，Hermes 产出的每个结果都成为你可以阅读、编辑、链接的笔记。

两个系统互相增益，而不是各自平行运转。

#  二、四层架构

整个系统有四层：

**第一层：笔记库层（Obsidian）**

纯文本 Markdown 文件，按同时适合人类浏览和自动化处理的结构组织。

**第二层：连接层（Filesystem MCP / Obsidian 原生 Provider）**

Hermes v0.14（2026 年 5 月 16 日发布）内置了 Obsidian Provider，一条命令就能连接：

```
hermes memory setup --provider obsidian --path ~/path/to/your/vaulthermes memory status
```

也可以通过 Filesystem MCP 实现读写访问。Hermes 能读取任何笔记、搜索所有笔记、创建新笔记、更新已有笔记。

**第三层：智能层（Hermes + Claude）**

Hermes 执行技能文件，通过 Anthropic API 调用 Claude。Claude 读取你的笔记库，跨积累的知识推理，产出输出并写回笔记库。

**第四层：自动化层（Hermes 调度器）**

定时工作流在设定时间自动触发：早晨你在醒来前收到基于笔记库生成的晨报；每周日晚上自动合成周回顾；每周一早上自动生成项目状态更新。一切都不需要你主动发起。

#  三、搭建步骤

**第一步：安装 Hermes Agent**

从 GitHub 克隆仓库，安装依赖。

**第二步：配置 Claude 作为模型**

在 .env 中配置 Anthropic 作为模型提供商，选择 Claude Opus 作为主力模型，填入 API Key。启用调度器。

**第三步：连接到 Obsidian 笔记库**

配置 Filesystem MCP 路径指向你的 Obsidian 笔记库根目录。或者使用 Hermes 原生的 Obsidian Provider。

这一步就是连接一切的关键。Hermes 现在可以直接访问你笔记库里的每一条笔记。

**第四步：安装 MCP 服务器**

安装 Filesystem MCP 和 Brave Search MCP，赋予 Hermes 本地文件读写和网络搜索能力。

**第五步：启动并验证连接**

启动 Hermes 后，让它描述你笔记库的顶层文件夹结构。如果它能准确描述，说明连接成功了。

#  四、笔记库结构设计

这个结构要同时满足人类导航和自动化处理的需求：

```
VAULT/00 - INBOX/ # 原始捕获，等待处理01 - NOTES/permanent/ # 原子笔记，用自己的话写daily/ # 每日笔记 [YYYY-MM-DD].mdmeetings/ # 会议笔记02 - PROJECTS/ # 活跃项目文件夹[project-name]/overview.mdtasks.mdnotes/03 - RESOURCES/ # 按主题分类的参考资料04 - HERMES-OUTPUTS/ # 所有自动化产出briefings/ # 晨报analyses/ # 分析报告syntheses/ # 周综合reviews/ # 项目健康报告05 - ARCHIVE/ # 已完成项目和过时材料06 - SYSTEM/ # CLAUDE.md 和技能文件
```

HERMES-OUTPUTS 文件夹是关键新增项。Hermes 所有自动化产出都放在这里，你始终知道去哪里找它产出的内容。

SYSTEM 文件夹放 CLAUDE.md 和技能文件参考。把系统说明放在笔记库里，意味着关于系统如何工作的说明和系统产出的知识在一起。

#  五、CLAUDE.md：系统的指挥中枢

CLAUDE.md 放在笔记库的 06 - SYSTEM/CLAUDE.md，Hermes 每次执行技能前通过 Filesystem MCP 读取它。

它需要包含：你是谁、你做什么、笔记库结构说明、活跃项目列表（每个一行状态 + 下一步行动）、本周优先级、你经常思考的话题、你的写作风格偏好。

还有两条关键指令：

记忆指令——每次技能执行前，同时读取 Hermes 数据库中的相关记忆和笔记库中的相关笔记，合并两个来源获取完整上下文。技能执行后，把摘要存回 Hermes 记忆并创建或更新笔记库中的相关笔记。

输出指令——生成任何输出前先读取笔记库的相关部分，用文件路径引用具体笔记，保存到 04 - HERMES-OUTPUTS 的正确子文件夹，用 YYYY-MM-DD-[类型]-[主题].md 格式命名，创建包含 type 和 date 的 frontmatter。

这个文件每周一审阅和更新，确保 Hermes 始终基于最新的上下文工作。

#  六、七个核心技能

## **技能 1：感知笔记库的晨报**

每天早上 6 点自动触发，基于你的真实笔记库生成晨报，而不是通用新闻。

流程：读取 CLAUDE.md 了解当前上下文 → 读取今天的日记 → 读取所有项目的 overview.md 了解当前状态 → 检查每个项目的 tasks.md 看是否有过期任务 → 读取上周的周回顾 → 用 Brave Search 查找与你当前优先级相关的外部动态 → 综合所有内容生成晨报。

晨报包含：今天最重要的事、每个项目一句话状态和下一步行动、昨天未完成的开放循环、与你优先级相关的外部信号、从你的笔记中发现的与今天的关联。

保存到 04 - HERMES-OUTPUTS/briefings/。

## **技能 2：收件箱处理器**

每天晚上 8 点自动清空你的 INBOX 文件夹。

流程：读取每个捕获的文件 → 分类（永久想法 / 项目笔记 / 参考资料 / 日记捕获 / 任务）→ 在笔记库中搜索相关的已有笔记 → 将文件归档到正确位置并创建链接 → 原始文件移到 ARCHIVE。

质量标准：每个新建的永久笔记必须至少有一条链接到已有笔记。没有连接的永久笔记还没准备好进入永久文件夹。

## **技能 3：项目健康监控**

每周一早上 7 点，读取所有活跃项目文件夹，产出健康评估。

对每个项目评估：状态（正常 / 有风险 / 停滞 / 阻塞）、证据（引用笔记库中的实际内容）、下一步行动（未来 48 小时最重要的事）、停滞标记（如果项目中任何文件 7 天以上未修改）。

只说能在具体笔记中追溯到的话，泛泛而谈的项目状态评估没有用。

## **技能 4：笔记连接发现器**

每周日晚上，读取过去七天创建或修改的所有笔记，发现与旧笔记之间尚未建立的连接。

对每条近期笔记：搜索所有永久笔记中的语义关联 → 找出尚未用 wikilink 建立的连接 → 评估连接强度（强 / 中 / 弱）→ 建议链接文本。

只上报强和中等强度的连接。价值在于发现人类读者会忽略的关联。

## **技能 5：周笔记综合**

每周日晚上综合整周的笔记库活动，产出一份只有跨七天积累才能看到的洞察报告。

流程：读取过去 7 天的所有日记、永久笔记、Hermes 产出、项目健康报告 → 综合所有内容。

报告包含：这一周一句话总结、推进了什么事、什么没有推进、涌现的想法、反复出现的模式、最重要的笔记连接、下周三个优先级、需要更新到 CLAUDE.md 的内容。

这份文档应该成为你周一早上第一个读的东西。

## **技能 6：研究转笔记转换器**

手动触发。把扔进 INBOX 的研究材料转成结构化的永久笔记。

流程：识别研究材料 → 提取核心观点、支撑证据、真正新颖的内容 → 搜索已有永久笔记 → 创建文献笔记（包含来源、核心论点、关键证据、个人反应、与已有笔记的连接）→ 从文献笔记中标记的想法创建新的永久笔记。

质量标准：每条文献笔记必须至少产出一条新的永久笔记。如果研究没有产生值得捕获为永久笔记的新想法，那这份研究本身就不值得处理。

## **技能 7：思考伙伴**

手动触发。让 Claude 作为活跃的思考伙伴，读取你的近期笔记，提出质疑，发现矛盾，推动你的思考前进。

流程：读取 CLAUDE.md 了解当前知识上下文 → 读取过去 14 天的永久笔记 → 读取过去 7 天的日记 → 发现有趣的张力、未充分展开的想法、缺失的连接、未回答的问题 → 对每一项进行深入追问。

语气应该像一个读过你所有笔记的聪明同事，在认真参与你的想法。不是附和，不是总结。是推动、质疑、连接。

#  七、运行一天是什么样的

早上 6 点：Hermes 读取你的笔记库生成晨报。你醒来时在 HERMES-OUTPUTS 里看到一份了解你项目状态、开放循环、当前优先级和相关外部动态的简报。五分钟读完，在看任何通知之前就知道今天什么最重要。

白天：你把所有东西捕获到 INBOX。灵感、任务、参考资料、会议笔记。不需要决定归档到哪里，只管捕获。

晚上 8 点：Hermes 自动处理你的收件箱。每个捕获都被归档、链接、连接到已有知识。INBOX 自动清空，没有未处理的内容。

周一早上 7 点：项目健康报告已在 HERMES-OUTPUTS 里。每个活跃项目被评估，停滞项目被标记，下一步行动已浮现。你带着对一切状况的完整了解开始新的一周。

周日晚上：连接报告和周综合依次到来。你的知识网络每周自动变密，不需要额外 effort。

#  八、90 天后的质变

这个组合的力量在第一天是看不见的。

它随着时间显现，因为两个独立的复利效应在交互。

Obsidian 的复利：你的永久笔记随着月数积累越来越多连接。一月写的笔记在三月、四月、五月被链接。网络越来越密，连接发现器涌现越来越不显然的关系。第三个月的笔记库比第一个月在质的层面更丰富。

Hermes 的复利：记忆跨所有会话积累。第九十天的晨报读过你三个月的日记，知道你的模式。第十二周的周综合有十二周的过往综合可以借鉴。第三个月的项目健康监控知道你项目通常会怎么停滞，在问题出现之前就把它标出来。

交互的复利：Obsidian 知道的越多，Hermes 推理得越好。Hermes 产出的越多，笔记库变得越丰富。笔记库越丰富，下一次 Hermes 产出质量越高。

三个月后，你拥有的不再是一个你使用的工具。

它像一个在你身边工作了好几个月的思考伙伴，读过你写的所有东西，涌出你自己工作记忆装不下的连接和洞察。

这就是 Obsidian + Hermes Agent 组合能做到的事。

这个周末就能开始搭建。

从第一个读取你笔记库的技能开始，复利就从那一刻启动。
